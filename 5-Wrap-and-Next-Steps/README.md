# 5. Wrap and Next Steps

Use this section to close the session and direct participants toward follow-up practice.

## Recap Questions

Ask participants to answer these before they leave:

1. When would you choose Azure DocumentDB over a relational database?
2. What is the difference between a `find()` query and an aggregation pipeline?
3. What does `COLLSCAN` tell you?
4. When would you use full text search instead of vector search?
5. Why is a pre-migration assessment important?

## Suggested Wrap-Up Messages

- MongoDB compatibility lowers migration friction, but assessment still matters.
- Search options are complementary: keyword, full text, and vector each solve a different problem.
- Indexing and query plans matter even in flexible-schema databases.
- MCP is most valuable when paired with strong database fundamentals.

## Recommended Next Steps

1. Re-run the query lab with your own sample collection.
2. Explore one more index design and compare the query plan.
3. Review document modeling patterns before designing a real workload.
4. Try the MCP demo on another collection and ask schema-focused questions.
